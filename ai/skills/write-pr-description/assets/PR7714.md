This PR lays the groundwork for offline caching within the `firebase-dataconnect` module by implementing mechanisms to efficiently store and retrieve complex query results. It defines new Protobuf structures to represent data entities and their hierarchical paths, along with utility functions to 'dehydrate' query results by pruning and separating entities, and 'rehydrate' them by grafting the entities back into the main structure. The changes are supported by comprehensive property-based tests to ensure the reliability of these new data serialization and deserialization processes.

There should be no side effects of this PR that are noticable to customers; this is merely internal code changes working towards offline caching support.

### Highlights

* **Offline Caching Logic**: Introduced core logic for dehydrating and rehydrating query results. This involves extracting entities from a main query result and storing them separately, then reconstructing the full result when needed.
* **New Protobuf Definitions**: Added new Protobuf messages in `sdk.proto` to support the caching mechanism, including `FieldOrListIndex`, `EntityPath`, `Entity`, `EntityList`, `EntityOrEntityList`, and `QueryResult`.
* **Data Manipulation Utilities**: Implemented `ProtoGraft` for inserting values into Protobuf `Struct`s and `ListValue`s at specified paths, and `ProtoPrune` for selectively removing descendants from Protobuf structures based on a predicate.
* **Path Handling Enhancements**: Added utility functions and comparators for `DataConnectPath` and `DataConnectPathSegment` to facilitate their use with the new Protobuf types and data manipulation logic.
* **Extensive Testing Utilities**: Introduced a suite of property-based testing utilities, including `QueryResultArb` for generating complex query result samples, and helpers for random insertion, walking, and custom Kotest printers for improved test readability and robustness.
