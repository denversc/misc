This PR wires up the offline caching mechanism in `firebase-dataconnect`. It integrates the SQLite cache database (introduced in previous PRs) into the main `executeQuery` flow. It handles cache lookups, insertions, and configuration based on `DataConnectSettings`. It also updates the Proto definitions to support GraphQL extensions for cache metadata.

Note that this plumbing effectively only enables query-level caching, as the implementation of `ExecuteQueryResponse.getEntityIdForPathFunction()` unconditionally returns `null`. Also, max_age is not implemented. A future PR will address these issues/limitations.

### Highlights

* **Cache Integration**: `executeQuery` now checks the local SQLite cache before making a network request. Valid cached results are returned immediately.
* **Cache Population**: Successful network responses are inserted into the cache.
* **Configuration**: The cache database name is deterministically calculated based on the connector configuration (Project ID, App Name, Connector ID, etc.) to ensure isolation.
* **Proto Updates**: Added `GraphqlResponseExtensions` to the `ExecuteQueryResponse` proto to carry cache-related metadata (like `max_age`, `entity_id`).
* **Proto Utilities**: Added utilities for calculating SHA-512 hashes with preambles and formatting Durations.
